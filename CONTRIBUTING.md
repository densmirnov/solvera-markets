# CONTRIBUTING

## Обязательные правила
1. Любые изменения в коде фиксируются в документации.
2. Любой новый или изменённый код покрывается тестами для минимизации ошибок.
3. Весь код проходит линтеры и автофиксаторы перед коммитом.
4. Все изменения проходят локальные проверки перед PR или релизом.

## Структура проекта
- `contracts/` — смарт‑контракты и тесты.
- `indexer/` — индексатор событий.
- `backend/` — API‑слой.
- `frontend/` — интерфейс оператора.
- `docs/` — проектная документация.
- `scripts/` — утилиты и проверки.
- `tests/` — кросс‑проектные тесты.

## Процесс изменения
1. Изменить код и тесты.
2. Обновить релевантные документы в `docs/`.
3. Запустить `make check`.
4. Зафиксировать изменения в PR.
